# Agent Artifact Checklist

- Is the trigger description precise enough that the skill fires only when it should?
- Is `SKILL.md` concise, with detailed material moved into `references/` instead of bloating the main file?
- Are commands, files, and workflows real and verifiable from the repo?
- Is the wording generic where the skill is meant to be reusable across PRs?
- Does the artifact separate offline calibration from online reasoning instead of telling the agent to cargo-cult old comments?
